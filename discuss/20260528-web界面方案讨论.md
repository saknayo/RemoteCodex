## 2026-05-28 19:00 - 流式消息显示与发送按钮时序修复（已解决）

### 用户需求
- "选中一个会话后，claude回复的内容没有显示，要切换会话再切回来才看到"
- "claude始终显示一个空白气泡，没有转圈也没有回复内容"
- "sending按钮变成send之后还等了很久才显示回复内容"

### 问题1：assistant消息未推送到前端

**现象：** 发送消息后只有空白气泡，切换会话再切回来才能看到内容。

**根因：** 服务端只 emit 了 userMsg 的 `message_added`，没有 emit assistantMsg 的。前端 `currentSession.messages` 最后一条是 user，`stream_chunk` 中 `lastMsg.role === 'assistant'` 判断失败，内容被丢弃。

**修复：** server.js 创建 assistantMsg 后立即 `socket.emit('message_added', assistantMsg)`。

### 问题2：stream_end 不携带完整内容

**现象：** 即使 stream_chunk 未实时更新 DOM，最终结果也应正确显示。

**修复：** `stream_end` 事件携带 `{ code, content: fullResponse }`，前端用服务端完整内容作为保底渲染。

### 问题3：发送按钮提前恢复

**现象：** 按钮在内容到达前就恢复为 "Send"，用户还要等几秒才看到内容。

**根因：** CLI 启动 3 秒后向 stderr 输出 stdin 警告：
```
Warning: no stdin data received in 3s, proceeding without it.
```
服务端将 stderr 作为 `stream_error` 发送给前端。前端 handler 将 `isStreaming=false` 并恢复按钮。但 CLI 仍在处理，stdout 内容数秒后才到达。

**时序分析（实测）：**
```
t=0ms    spawn CLI
t=3321ms stderr 警告 → stream_error → 按钮恢复（错误！）
t=6401ms stdout 输出 → stream_chunk（但 streamingContentEl 已被清空）
t=6423ms close → stream_end → 内容才显示
```

**修复：** `stream_error` 不再终止流式传输。stderr 警告不影响 UI 状态，只有 `stream_end`（CLI 真正结束）才恢复按钮和显示内容。

### 修改文件
- `server.js`：assistantMsg 添加 message_added emit；stream_end 携带 content；添加 DEBUG 日志
- `public/app.js`：streamingContentEl DOM 引用方式；stream_end 直接更新最后一个气泡 DOM；stream_error 不终止流式
- `public/index.html`：版本号更新至 v31

### 经验教训
- CLI 的 stderr 不等于错误，可能只是警告/日志，不应影响业务逻辑
- 调试时序问题应先实测底层工具（spawn CLI）的实际输出时序，而非猜测

## 2026-05-28 18:00 - 聊天区域滚动问题（已解决）

### 用户需求
- 继续修复聊天区域滚动问题
- "登录进去之后是空白的，只有最上面的导航栏"

### 根因分析

经过 v7-v21 共 15 次尝试未解决，最终定位到两个核心问题：

1. **高度链条断裂**：`html/body(100%) → #app(缺失 height:100%) → .view(100%) → .container(flex:1)`
   - `#app` 没有设置高度，导致所有子元素的百分比高度和 flex 分配失效
   - 这也是之前所有 flex/grid/min-height:0 方案都不生效的根本原因

2. **容器溢出未控制**：`.container` 和 `.chat-area` 缺少 `overflow: hidden`
   - 没有阻止子元素撑开父容器，导致滚动区域无法正确收缩

### 最终修复（CSS v=22 → v=23）

```css
/* 1. 补全高度链条 */
#app { height: 100%; }

/* 2. container 添加溢出控制 */
.container {
  overflow: hidden;
}

/* 3. chat-area 添加溢出控制和 min-width */
.chat-area {
  min-width: 0;
  overflow: hidden;
}

/* 4. messages 溢出改为 auto（无内容时不显示滚动条） */
.messages {
  overflow-y: auto;
}
```

同时移除了之前调试用的蓝色/红色诊断边框，给 app.js 也加上了版本号参数避免缓存问题。

### 经验教训
- Flex 嵌套布局的高度问题，首先要检查从根到叶的完整高度链条是否通畅
- `min-height: 0` 只有在高度链条完整的前提下才有效
- 遇到反复修改不生效时，应优先排查是否遗漏了中间层容器的高度设置
