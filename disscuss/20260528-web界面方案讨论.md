## 2026-05-28 13:30 - 加入登录间隔限制

### 用户需求
- "每次登录间隔30秒"

### 讨论结论

加入登录间隔限制，更新 SECURITY_CONFIG：
```javascript
const SECURITY_CONFIG = {
  LOGIN_INTERVAL: 30,           // 最小登录间隔（秒）
  MAX_ATTEMPTS: 3,              // 最大失败次数
  BLOCK_DURATION: 30 * 60,      // 封禁时长（秒），30分钟
  ATTEMPT_WINDOW: 15 * 60,      // 尝试计数窗口（秒），15分钟
  RATE_LIMIT_WINDOW: 60,        // 速率限制窗口（秒）
  RATE_LIMIT_MAX: 10,           // 窗口内最大请求数
  CLEANUP_INTERVAL: 60 * 60     // 清理过期数据间隔（秒），1小时
};
```

前端需显示倒计时等待界面。

## 2026-05-28 13:10 - 初始需求沟通

### 用户需求
1. 技术栈：稳定的，无明显偏好
2. 功能：简单对话、会话历史、会话内容渲染
3. 部署：个人使用、密码保护、公网、服务器在内网但有公网跳板
4. 环境：Linux、命令行调用CLI

### 讨论结论

**推荐方案：** Node.js + Express + Socket.io
- 与 Claude CLI 同栈，兼容性好
- WebSocket 支持流式输出
- 简单稳定

**功能设计：**
- 密码认证（后升级为用户名+密码）
- 会话管理
- 流式输出
- 内容渲染

**部署架构：**
```
公网跳板机(Nginx) → 内网服务器(Web Server + CLI)
```
